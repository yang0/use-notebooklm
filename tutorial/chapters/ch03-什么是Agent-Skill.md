# 第三章：什么是 Agent Skill——定义、格式与渐进式披露

## Skill 的准确定义

> **Skill = 为 Agent 打包的领域专业知识的有组织文件集合**

用更简单的话说：**Skill 就是一个文件夹**。

这种"极简主义"是刻意设计的。Anthropic 希望：
- ✅ **任何人**都能创建（不只是开发者）
- ✅ **任何 Agent**都能使用
- ✅ **兼容现有工具**：Git 版本控制、Google Drive 存储、zip 分享

---

## Skill 的物理形态

一个 Skill 本质上是一个文件夹，结构如下：

```
my-skill/
├── skill.md          # 核心指令和目录（必需）
├── scripts/          # 可执行脚本（工具）
│   ├── tool_a.py
│   └── tool_b.sh
├── templates/        # 模板文件
├── assets/           # 资源文件（图片、数据等）
└── docs/             # 补充文档
```

### skill.md 的作用

`skill.md` 是每个 Skill 的核心，包含：

1. **元数据（Metadata）**：Skill 的名称、描述、用途
2. **核心指令（Core Instruction）**：告诉 Agent 如何使用这个 Skill
3. **目录（Directory）**：文件夹结构的说明
4. **脚本说明**：每个脚本的用途和参数

---

## 渐进式披露（Progressive Disclosure）

这是 Skills 架构中**最关键的设计**之一。

### 问题：Context Window 有限

如果 Agent 同时掌握数百个 Skill，把所有 Skill 的详细内容都塞进上下文窗口：
- ❌ 上下文窗口被撑爆
- ❌ 每个 Skill 都占用宝贵的 token 预算
- ❌ 实际可用上下文大幅减少

### 解决方案：只展示元数据

Skills 采用"渐进式披露"机制：

**阶段一：初始展示**
- Agent 只看到所有 Skill 的**元数据**
- 元数据非常精简：名称 + 一句话描述
- 占用极少 token

**阶段二：按需加载**
- 当 Agent 决定使用某个 Skill 时
- 才会读取该 Skill 的 `skill.md` 完整内容
- 获得核心指令和目录

**阶段三：执行时加载**
- 只有当需要执行具体脚本时
- 才会读取脚本文件本身
- 执行完毕后释放相关上下文

### 类比：图书馆索引系统

想象 Agent 是一座图书馆：
- **传统方式**：每次把所有书的内容都摊在桌上（不可行）
- **Skills 方式**：桌上只有**索引卡片**（元数据），需要哪本书再去**书架**取（按需加载）

这样 Agent 可以同时"掌握"数千个 Skill，但任何时候只加载真正需要的那个。

## 为什么用文件夹？

Barry 和 Mahesh 反复强调：**"We like files"（我们喜欢文件）**。

### 文件的优势

| 优势 | 说明 |
|------|------|
| **通用性** | 文件是计算机最基础的原语，任何系统都支持 |
| **版本控制** | Git、SVN 等工具天然支持 |
| **可分享** | Zip、云盘、邮件，随便怎么传 |
| **可读性** | 人类可以直接打开查看和编辑 |
| **可修改** | Agent 和人类都能修改 |
| **持久化** | 保存在文件系统里，不会丢失 |

### 可组合性

因为 Skill 是文件夹，所以：
- 可以**嵌套**：一个 Skill 可以引用另一个 Skill
- 可以**合并**：把多个 Skill 打包在一起
- 可以**分发**：像分发软件一样分发 Skill

## Skill 的实际例子

### 例子1：幻灯片样式脚本

演讲中提到一个具体场景：

> "我们发现 Claude 反复写同一个 Python 脚本来给幻灯片应用样式。于是我们让 Claude 把这个脚本保存到 Skill 里，作为工具给'未来的自己'使用。"

现在 Claude 只需要运行这个脚本，就能一致、高效地完成样式应用。

### 例子2：文档 Skill

Anthropic 自己发布的 Skill：
- 让 Claude 能够**创建和编辑专业级办公文档**
- 包含文档格式规范、模板、常用操作脚本

### 例子3：科学研究 Skill

由 Cadence 构建：
- EHR（电子健康记录）数据分析
- 常用 Python 生物信息学库的使用指南
- 让 Claude 在生物医学领域表现更专业

## Skill vs 传统 Prompt

| 维度 | 传统 Prompt | Skill |
|------|-------------|-------|
| 持久性 | 一次性，对话结束就消失 | 文件形式，永久保存 |
| 复用性 | 每次重新输入 | 一次创建，多次使用 |
| 共享性 | 复制粘贴 | Git、云盘、zip 分享 |
| 可修改 | 需要重新写 | 直接编辑文件 |
| 版本控制 | 无 | Git 完整支持 |
| 上下文占用 | 全部在 prompt 里 | 渐进式加载 |

---

## 总结

Skill 的设计哲学是**"极致简单，极致强大"**：

1. **简单到只是一个文件夹**
2. **强大到可以包含任何专业知识**
3. **聪明到只加载需要的部分**
4. **开放到任何人都能创建**

这种设计让 Skill 不只是一个功能，而是一个**平台**——一个让 Agent 能力无限扩展的平台。
