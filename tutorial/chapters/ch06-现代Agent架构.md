# 第六章：现代 Agent 架构——四层堆栈

## 架构演进

Barry 和 Mahesh 提出，通用 Agent 的架构正在**收敛**到一个标准化的四层堆栈。这不是理论推测，而是 Anthropic 在实际部署 Claude 时验证的架构。

---

## 四层堆栈详解

### 第一层：Agent Loop（代理循环）

**作用**：管理模型的内部上下文

**具体职责**：
- 控制哪些 token 进入模型
- 控制哪些 token 从模型输出
- 管理对话历史和任务状态
- 决定何时停止、何时继续

**类比**：就像操作系统管理 CPU 调度，Agent Loop 管理模型的"注意力"和"工作流"。

### 第二层：Runtime Environment（运行时环境）

**作用**：给 Agent 一个"数字工作空间"

**提供的能力**：
- ✅ **文件系统**：读写文件、创建文件夹、组织数据
- ✅ **代码执行**：运行 Python、bash、JavaScript 等
- ✅ **进程管理**：启动和协调多个任务

**核心洞察**：

演讲者强调，当 Agent 拥有文件系统和代码执行能力后，核心脚手架可以薄到只剩下 **bash + 文件系统**。

**类比**：Runtime 是 Agent 的"操作系统"，给它提供了与数字世界交互的基本能力。

### 第三层：MCP Servers（模型上下文协议服务器）

**作用**：连接 Agent 与外部世界

**提供的能力**：
- 连接数据库
- 调用外部 API
- 访问文件系统
- 与第三方服务交互

**关键特性**：
- MCP 已成为 Agent 连接性的**行业标准**
- 标准化了 Agent 与外部工具的通信协议
- 让 Agent 能够获取实时数据和外部资源

**类比**：MCP 是 Agent 的"网络接口"，让它能够访问互联网和企业内部系统。

### 第四层：Skill Library（技能库）

**作用**：提供按需加载的专业知识

**特性**：
- 包含数百甚至数千个 Skills
- 每个 Skill 是一个文件夹，包含专业知识和脚本
- 采用渐进式披露：只在使用时加载

**类比**：Skill Library 是 Agent 的"应用程序库"，每个 Skill 是一个专业应用。

---

## 四层如何协同工作

让我们用一个具体场景来说明四层如何协作：

### 场景：生成一份财务报告

**Step 1: Agent Loop 启动**
- 用户要求"生成 Q3 财务报告"
- Agent Loop 接收任务，开始规划

**Step 2: Runtime 提供工作空间**
- Agent 在文件系统创建 `q3_report/` 文件夹
- 准备存储数据和最终报告

**Step 3: MCP 获取外部数据**
- 通过 MCP 连接财务数据库
- 拉取 Q3 的财务数据
- 获取市场研究数据

**Step 4: Skill 提供专业指导**
- Agent 识别到需要"财务报告 Skill"
- 加载该 Skill 的指令和脚本
- 使用 Skill 中的 Python 脚本分析数据
- 按照 Skill 中的模板生成报告

**Step 5: 输出结果**
- 报告保存到文件系统
- Agent Loop 确认任务完成
- 向用户呈现结果

## "Agent + MCP + Skills" 部署模式

Anthropic 用这个模式**快速部署 Claude 到垂直行业**：

### 案例：金融服务和生命科学

发布 Skills **仅 5 周后**，Anthropic 立即推出了：

#### 金融服务版 Claude
- **MCP 连接**：金融数据库、市场数据、监管数据源
- **Skills 库**：
  - 财务报表生成
  - 合规检查流程
  - 风险评估模型
  - 投资组合分析

#### 生命科学版 Claude
- **MCP 连接**：医学数据库、临床试验数据、基因组数据
- **Skills 库**：
  - EHR 数据分析
  - 生物信息学分析
  - 药物研发流程
  - 临床文档处理

### 关键洞察

> **不需要重新造 Agent，只需要装备不同的 MCP + Skills。**

这是"通用 Agent + 模块化扩展"架构的威力：
- 底层 Agent 完全相同
- 差异化来自 MCP 连接的数据源
- 差异化来自 Skills 提供的专业知识

---

## 架构的核心优势

### 1. 可组合性

- 任意组合 MCP Servers
- 任意组合 Skills
- 为不同场景快速组装定制 Agent

### 2. 可扩展性

- 新增能力只需添加新的 MCP 或 Skill
- 不需要修改 Agent 核心
- 生态系统可以无限增长

### 3. 可维护性

- 每个组件独立更新
- Skills 可以版本控制
- MCP 可以独立升级

### 4. 可预测性

- Agent 行为由明确的 Skill 定义
- 不依赖模型的"涌现能力"
- 输出更稳定、更可靠

---

## 与计算机架构的类比

演讲者用这个类比来总结整个架构：

| AI 层 | 计算机层 | 说明 |
|-------|----------|------|
| **模型 (Models)** | **处理器 (Processors)** | 需要巨额投资，蕴含巨大潜力，但本身用途有限 |
| **Agent Runtime** | **操作系统 (OS)** | 编排资源、进程和数据，让处理器更有价值 |
| **MCP Servers** | **网络/驱动** | 连接外部世界，获取数据和资源 |
| **Skills** | **应用程序** | 解决具体问题，编码领域专业知识 |

> **正如操作系统让处理器更有价值，Agent Runtime 让模型更有价值。**
> 
> **正如应用程序让操作系统真正有用，Skills 让 Agent 真正有用。**

---

## 总结

现代 Agent 架构正在从"每个场景造新 Agent"转向"一个通用 Agent + 模块化扩展"。

四层堆栈各司其职：
- **Agent Loop**：管理智能
- **Runtime**：提供工作空间
- **MCP**：连接世界
- **Skills**：提供专业

这种架构让 Agent 从"通用但无能"变成了"通用且专业"——这才是 Agent 真正走向生产环境的关键。
