# 第一章：引言——Agent 的新范式

## 演讲背景

这篇演讲来自 Anthropic 的两位核心成员 **Barry Zhang** 和 **Mahesh Murag**。他们不是理论家，而是 Claude Skills 功能的创造者。演讲时距离 Skills 发布仅 **5 周**，但生态系统中已经涌现出 **数千个 Skills**。

演讲的核心命题非常直接：**"我们停止造 Agent，开始造 Skill"**（We stopped building agents and started building skills instead）。

---

## 从"造Agent"到"造Skill"的转变

### 过去：每个领域一个Agent

在过去的一年里，AI 行业的主流思路是：
- 写代码需要 **Coding Agent**
- 做研究需要 **Research Agent**
- 写文档需要 **Writing Agent**
- 每个垂直领域都需要重新造一个 Agent

每个 Agent 都有自己的工具链、脚手架（scaffolding）和架构，导致开发和维护成本极高。

### 现在：一个通用Agent + 无数Skill

Barry 和 Mahesh 的核心洞察是：**Agent 的底层远比我们想象得更通用**。

当我们把 Agent 与运行时环境（文件系统 + bash + 代码执行能力）紧密耦合后，突然发现：
- **生成财务报告**：Agent 可以调用 API 拉数据 → 用文件系统组织数据 → 用 Python 分析 → 用代码合成洞察
- **编辑文档**：Agent 可以读取文件 → 用脚本修改格式 → 保存结果
- **网页自动化**：Agent 可以用 Stagehand 操作浏览器

所有这些，**底层都是同一个 Agent**，区别只在于它掌握了什么领域的知识——也就是 Skill。

### 为什么"代码就是一切"

演讲者用一句话概括了新范式：

> **"We think code is all we need."**

在数字世界里，代码是通用接口。只要 Agent 能读写代码、操作文件系统、执行 bash 命令，它就能完成几乎所有数字任务。剩下的唯一问题不是"怎么让 Agent 做"，而是"让 Agent 知道怎么做"——这正是 Skill 要解决的核心问题。

---

## 新范式的三层含义

### 1. Agent 变得更薄

以前每个 Agent 都需要厚重的自定义脚手架。现在核心可以薄到只剩下：
- **Agent Loop**（管理模型上下文）
- **Runtime**（文件系统 + 代码执行）

### 2. 差异化上移

真正的差异化不再是"Agent 本身"，而是：
- **MCP Servers** 提供外部数据连接
- **Skills** 提供专业知识和操作流程

### 3. 生态系统爆发

正因为底层统一，上层可以百花齐放：
- 开发者造第三方 Skill
- 企业内部造团队 Skill
- 非技术人员也能造专业 Skill

本次演讲围绕以下主线展开：
