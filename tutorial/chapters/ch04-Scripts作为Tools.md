# 第四章：Scripts 作为 Tools——为什么脚本比传统 AI Tool 更好

## 传统 AI Tools 的问题

演讲者明确指出，传统的 AI "工具"（Tools）存在几个根本性问题：

### 问题1：指令质量差

很多工具的说明写得很差：
- ❌ 描述模糊
- ❌ 参数定义不清
- ❌ 使用场景不明确

结果是：模型不知道该在什么时候用什么参数调用工具。

### 问题2：无法修改

当工具表现不好时：
- ❌ 模型无法修改工具本身
- ❌ 只能不断尝试不同的调用方式
- ❌ 陷入"代码启动问题"（cold start problem）

### 问题3：永远占着上下文

传统工具一旦定义，就**一直待在上下文窗口里**：
- ❌ 即使当前任务不需要这个工具
- ❌ 也持续消耗 token 预算
- ❌ 挤占了真正需要的信息空间

---

## Scripts 作为 Tools 的优势

Skills 用**脚本（Scripts）**来替代传统工具，解决了上述所有问题：

### 优势1：自文档化（Self-documenting）

脚本本身就是代码：
- ✅ 代码逻辑清晰，不需要额外解释
- ✅ 参数类型在代码中明确定义
- ✅ 用法可以通过阅读代码直接理解

### 优势2：可修改（Modifiable）

当脚本表现不好时：
- ✅ Agent 可以直接修改脚本代码
- ✅ 调整逻辑、参数、输出格式
- ✅ 修改后的版本保存下来，下次直接使用

### 优势3：存文件系统（Filesystem-based）

脚本存储在文件系统中：
- ✅ 不需要时完全不占用上下文
- ✅ 需要时才加载到上下文
- ✅ 执行完毕即可释放

---

## 对比总结

| 维度 | 传统 Tool | Script |
|------|-----------|--------|
| 文档质量 | 依赖人工写的说明 | 代码即文档 |
| 可修改性 | ❌ 不可修改 | ✅ 可直接编辑 |
| 上下文占用 | ❌ 始终占用 | ✅ 按需加载 |
| 持久化 | 内嵌在系统里 | 存文件系统 |
| 版本控制 | ❌ 通常不支持 | ✅ Git 完全支持 |
| 调试难度 | 黑盒 | ✅ 可阅读、可修改 |

---

## 实际例子：幻灯片样式脚本

演讲中提到的具体案例：

### 问题

Claude 在帮助用户制作幻灯片时，反复写同一个 Python 脚本：
- 应用字体样式
- 调整颜色方案
- 统一布局格式

每次都要重新写，既低效又不一致。

### 解决

Anthropic 让 Claude 把这个脚本保存到 Skill 里：

```python
# slides_styling.py
# 保存在 skill/scripts/ 目录下

def apply_style(slides, theme="corporate"):
    """Apply consistent styling to presentation slides."""
    for slide in slides:
        slide.font = THEMES[theme]["font"]
        slide.colors = THEMES[theme]["colors"]
        slide.layout = THEMES[theme]["layout"]
    return slides
```

### 效果

- ✅ **一致性**：每次样式完全相同
- ✅ **效率**：直接运行脚本，不需要重写
- ✅ **可进化**：需要新样式时，直接修改脚本
- ✅ **可共享**：团队成员都可以使用

## 更深层的意义

Scripts 作为 Tools 不仅仅是"工具形式的改变"，而是代表了**Agent 与工具关系的重构**：

### 从"调用"到"拥有"

- **传统**：Agent 调用外部工具（API 调用）
- **Skills**：Agent 拥有并管理自己的工具（文件系统里的脚本）

### 从"消费"到"创造"

- **传统**：Agent 只能使用预定义的工具
- **Skills**：Agent 可以自己创造新工具（写脚本保存到 Skill）

### 从"黑盒"到"白盒"

- **传统**：工具是黑盒，出问题只能猜
- **Skills**：脚本是白盒，可以阅读、修改、调试

---

## 对开发者的启示

如果你是 Agent 开发者或使用者，应该：

1. **优先使用脚本**：把常用操作写成脚本，存到 Skill 里
2. **让 Agent 自己写工具**：遇到重复操作时，让 Agent 写脚本保存
3. **建立工具库**：积累常用脚本，形成自己的 Skill 库
4. **版本管理**：用 Git 管理脚本，追踪变化

---

## 总结

> **传统 Tools 是"借用"，Scripts 是"拥有"。**
> 
> 当 Agent 拥有了自己的工具库，它才能真正成为领域专家。

Scripts 作为 Tools 是 Skills 架构中看似简单但影响深远的设计。它让 Agent 从"工具的使用者"变成了"工具的创造者和所有者"，这是 Agent 能力跃迁的关键一步。
